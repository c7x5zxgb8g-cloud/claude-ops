这是一次性的项目熟悉流程。**目标不是"记在脑子里"，而是把发现写到磁盘**——因为会话上下文会被 /clear 清掉，只有落盘的才算数。

请探索本仓库，查清以下事项，然后把结论**覆盖式写入** `ops-state/project-facts.md`：

1. 这是什么系统：有哪些服务 / 模块，用什么语言和框架。
2. 日志源清单（多 agent 系统重点）：把**所有**日志文件枚举出来，逐条记录路径或通配符、对应哪个 agent / 服务、格式（JSON / 纯文本）、轮转方式、以及怎么取最新内容（如 tail 多少行、按时间过滤）。这是会随项目变化的清单，也是以后日志位置变动时你要更新的地方。
3. 如何判断健康：关键错误特征、典型异常堆栈样子、有没有现成的健康检查或指标端点。
4. 测试怎么跑：命令、所需环境。
5. 代码改动应限制在哪些目录（给 `.claude/hooks/guard.sh` 的 `ALLOWED_PREFIXES` 提建议）。

`ops-state/project-facts.md` 结构：

    # 项目实况（由 /onboard 生成，可重跑刷新）
    ## 系统概览
    ## 日志源清单（逐条：路径/通配符 · 哪个 agent · 格式 · 如何取最新）
    ## 健康判断信号
    ## 测试命令
    ## 建议的 allowed_paths

写完后，简要汇报发现，并指出 `guard.sh` 的 `ALLOWED_PREFIXES`、`CLAUDE.md` 里的日志路径是否需要按上面调整——**由我手动改**，你不要动这两个文件。
